# Demo 知识库

本目录为 RAG 知识库问答系统的示例文档，可直接上传到系统用于体验检索与问答。

## 文件说明

| 文件 | 内容概要 |
|------|----------|
| 人事制度.md | 年假、考勤、报销、人事联系方式 |
| 产品说明.md | 产品定位、支持平台、文档格式、核心功能与版本 |
| 开发规范.txt | 分支策略、提交规范、接口安全、测试与上线 |
| 常见问题.md | 账号登录、文档上传、检索与问答、技术支持 FAQ |
| 订单与退款政策.md | 国内/海外订单退款期限、跨境说明、对照表（多轮指代演示） |
| 退款流程与申请表.md | 退款三步流程、第二步材料、A/B/C 类申请表（与文档 §1.5 例 A/B 配套） |

## 如何使用

1. **启动服务**：确保 MySQL、后端与（可选）前端已按项目 README 启动。
2. **上传文档**：通过管理端接口上传本目录下的文件。
   - 接口：`POST /api/admin/documents`
   - Content-Type: `multipart/form-data`，字段名：`file`（具体以服务端约定为准）
   - 可用 curl 示例（在项目根目录执行）：
     ```bash
     curl -X POST http://127.0.0.1:7001/api/admin/documents \
       -F "file=@demo-knowledge-base/人事制度.md"
     ```
3. **等待处理**：上传后系统会异步完成清洗、分块与向量化，稍等再提问。
4. **体验问答**：调用 `POST /api/qa`，body `{ "question": "公司年假几天？" }` 等，即可得到带引用的答案。

## 推荐试问示例

- 公司年假几天？年假上限多少？
- 产品支持哪些文档格式？单文件多大？
- 分支策略是什么？提交信息格式怎么写？
- 忘记密码怎么办？支持 SSO 吗？

### 多轮示例（与《多轮知识库对话》文档 §1.5 对照）

**先分清两件事**（避免和「表格是不是用户手打进提问里」搞混）：

1. **【参考内容】**：来自**本轮检索**命中的 chunk，系统在后台拼进 prompt；用户输入里**没有**粘贴表格全文。  
2. **对话历史**：上一轮 user/assistant 的**聊天记录**，用来接话、消解「你刚才说的」「第二步」等指代。

---

#### 例 A：说明「为什么要改写再 embedding」（检索召回）

**请先上传** `退款流程与申请表.md`，等待索引完成。知识库「第一节」写明了第一步、**第二步须上传哪些材料**、第三步。

在同一会话内按顺序提问（第 1 轮用完整句，第 2 轮故意用短句）：

| 轮次 | 角色 | 示例（可照抄） |
|------|------|----------------|
| 1 | user | 退款线上申请流程分几步？每一步大概做什么？ |
| 1 | assistant | （由系统根据检索生成；应概括出第一步、第二步材料、第三步等。） |
| 2 | user | **第二步要准备什么？** |

**说明**：第 2 轮只有「第二步…」，若**不做改写**、直接把该句拿去 embedding，容易和文档里的长表述对不齐。**改写**应结合上一轮 assistant 已列步骤，得到类似「退款流程第二步需要上传哪些材料」的句子，再 embed，更容易命中正确片段。本例主要体现**检索**；若作答也用这句改写结果且 chunk 已写全，有时只靠参考块也能答对事实。

---

#### 例 B：说明「为什么生成答案仍要带对话历史」（锚定助手口播）

仍使用已上传的 `退款流程与申请表.md`。知识库「第二节」分条写了 **A 类（境内）**、**B 类（境外）**，另有 **C 类（企业）**，文档中存在**多种申请表**，检索可能命中多段。

| 轮次 | 角色 | 示例（可照抄） |
|------|------|----------------|
| 1 | user | 境内收货和境外收货退款时，用的申请表是不是同一种？ |
| 1 | assistant | （由系统生成；与知识库一致时应类似：**不是同一种；境内用 A 类，境外用 B 类**。） |
| 2 | user | **那我走境外的话，你刚才说用哪个表？** |

**说明**：用户问的是「**你上一句归纳里**境外对应哪一种」，而不只是「文档里境外用哪类表」。仅把改写检索句 + 【参考内容】交给模型、**不带上一轮 assistant** 时，也可能答「B 类」，但在多表并存时更容易罗列或偏离**本轮口播**；**生成侧带上 assistant 原文**更稳。
