## 1. 项目与依赖

- [x] 1.1 创建项目结构（包/模块划分：文档处理、向量存储、检索匹配、答案生成）
- [x] 1.2 添加依赖：向量库客户端、嵌入/LLM 调用 SDK，以及配置加载方式
- [x] 1.3 定义配置项：向量库（LanceDB）路径、嵌入模型/端点、LLM 端点、chunk 是否按标题切分、最大长度、重叠大小

## 2. 文档处理

- [x] 2.1 实现文档上传接口：接收文件或 URL，持久化原始文档到 MySQL 文档表（doc_id、文件名、上传时间、状态等）
- [x] 2.2 实现文本清洗：从文档提取文本，规范化空白与编码，产出清洗后全文
- [x] 2.3 实现文本拆分：第一段可选按标题/结构（Markdown 等）切大块，第二段按段落/句子 + 最大长度 + 重叠切分；片段写入 MySQL 片段表（chunk_id、doc_id、text、可选 metadata）
- [x] 2.4 串联流水线：上传完成后自动触发清洗与拆分，产出可供向量存储消费的 chunk 列表

## 3. 向量存储

- [x] 3.1 实现 EmbeddingProvider 抽象及至少一种后端（本地或 OpenAI-compatible API）
- [x] 3.2 接入 LanceDB（vectordb）：仅写入向量与 chunk_id（不存原文），数据持久化到本地目录
- [x] 3.3 实现"文本片段 → 向量生成完成"流程：消费文档处理产出的 chunk，调用 EmbeddingProvider 并写入 LanceDB
- [x] 3.4 实现按 query 向量 top-k 检索接口，返回 [{ chunk_id, score }, ...]

## 4. 检索匹配

- [x] 4.1 实现检索请求接收：用户问答接口入参校验（必填 question），记录检索请求
- [x] 4.2 将用户 query 转为 embedding，调用向量存储 top-k 检索，取回 chunk_id 与分数
- [x] 4.3 根据 chunk_id 从 MySQL 片段表取原文、关联文档表取文件名等，组装为"匹配到的内容"列表（chunk_id、doc_id、text、score、文档名等），按 score 降序供答案生成使用
- [x] 4.4 无 chunk 时返回空列表并可选提示"知识库为空"

## 5. 答案生成

- [x] 5.1 实现 LLMProvider 抽象及至少一种后端（本地或远程 LLM API）
- [x] 5.2 实现提示编排：System 约束"仅根据【参考内容】回答、无法得到答案时说明不编造"；User 为【参考内容】（匹配内容格式化并标注 doc/chunk 来源）+ 用户问题；调用 LLM 生成答案
- [x] 5.3 实现答案生成完成流程：返回 answer 文本，并附带引用列表（文档名、chunk 等）
- [x] 5.4 无检索结果时返回友好提示（如知识库为空），不调用 LLM 编造内容

## 6. API 与集成

- [x] 6.1 管理员 REST API：文档上传、文档列表、文档删除（可选）
- [x] 6.2 用户 REST API：问答接口（请求 question，返回 answer + 引用片段）
- [x] 6.3 端到端联调：上传文档 → 自动清洗/拆分/写入 MySQL 与 LanceDB → 用户提问 → 检索 → 取原文 → 生成答案并带引用
- [x] 6.4 为上传与嵌入/LLM 调用增加超时与简单重试；答案生成失败时兜底（如仅返回检索到的片段）

## 7. 前端（React 18 + TypeScript + Rsbuild + Ant Design）

- [x] 7.1 创建前端项目：使用 Rsbuild 初始化 React 18 + TypeScript 项目，接入 Ant Design，配置开发时代理或 API baseURL 指向后端（如 http://127.0.0.1:7001）
- [x] 7.2 文档管理页布局与上传：路由 `/admin/documents`，页面标题「知识库文档管理」；上方操作区使用 Upload 组件支持点击/拖拽上传，支持格式与后端一致（如 .txt、.md、.pdf 等），上传前做格式与大小校验（与后端限制一致）、上传中 loading 并禁止重复提交，上传成功/失败用 message 提示并刷新列表
- [x] 7.3 文档管理页列表与删除：使用 Table 展示文档列表，列包括文件名、上传时间、状态（处理中/成功/失败）、操作（删除）；支持分页（首版可为固定每页条数或「加载更多」）；无数据时显示空状态（如「暂无文档，点击上方上传添加」）；删除前 Modal 二次确认，确认后调用删除 API，成功 message 并刷新列表，失败展示错误信息；若后端支持，处理中状态可轮询更新为成功/失败
- [x] 7.4 问答页布局与提问：路由 `/qa`（`/` 同样指向问答页），页面标题「知识库问答」；主内容区为 Input.TextArea（占位符「请输入您的问题」）+ 提交按钮；首次进入时回答区不展示或展示引导文案（如「在上方输入问题开始提问」）；提交前校验问题非空（空时禁用提交或 toast 提示），提交中按钮 loading、输入区禁用防重复提交
- [x] 7.5 问答页回答与引用：提交成功后展示答案正文（answer，若接口返回 Markdown 则做简单 Markdown 渲染）及引用区块（references：文档名、chunk 来源等，可折叠/展开）；无检索结果时展示接口返回的提示文案（如「知识库暂无相关内容」），不展示答案区块；网络或服务错误时 message 提示并允许用户重试
- [x] 7.6 前端与后端联调：确认文档上传/列表/删除、问答接口的请求与响应格式与后端一致，首屏与空状态、错误态展示正确
