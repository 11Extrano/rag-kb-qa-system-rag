# 产品页面与功能说明

本文档定义首版前端的页面清单、每页布局与主要功能细节，与 proposal / design 中的能力与 API 对应。前端技术栈：React 18 + TypeScript + Rsbuild + Ant Design。

---

## 页面清单

| 页面 | 路由（示例） | 目标用户 | 对应 API |
|------|----------------|----------|----------|
| 文档管理页 | `/admin/documents` 或 `/documents` | 系统管理员 | 文档上传、文档列表、文档删除 |
| 问答页 | `/` 或 `/qa` | 用户 | 问答接口（question → answer + references） |

首版可不做登录与权限，通过路由或入口区分管理员与普通用户；后续可加鉴权。

## 1. 文档管理页

### 1.1 目标

系统管理员在此上传文档、查看已上传文档列表、删除文档。上传后系统自动执行清洗、拆分、向量化（见 design 流水线）。

### 1.2 页面结构

- **顶部**：页面标题，如「知识库文档管理」。
- **上方操作区**：
  - 上传入口：支持点击或拖拽上传文件（单次可单文件或多文件，依实现而定）。
  - 支持格式需与后端一致（如 .txt、.md、.pdf 等），不支持格式在提交时校验并提示。
- **下方列表区**：
  - 表格展示已上传文档列表。
  - 列：文件名、上传时间、状态（处理中 / 成功 / 失败）、操作（删除）。
  - 支持分页（可选，首版可简单「加载更多」或固定每页条数）。

### 1.3 主要功能细节

| 功能 | 细节 |
|------|------|
| 上传 | 选择文件后调用上传 API；上传中显示 loading，禁止重复提交。 |
| 格式校验 | 前端可做基础校验（如后缀）；不支持格式时提示明确错误信息（与 spec 一致）。 |
| 列表 | 进入页面或上传成功后请求文档列表 API，表格展示；无数据时显示空状态文案（如「暂无文档，请上传」）。 |
| 删除 | 行内「删除」按钮，点击后二次确认（如 Modal），确认后调用删除 API；成功用 message 提示并刷新列表，失败提示错误原因。 |
| 状态 | 状态列展示当前文档处理状态；若后端支持，可轮询或 WebSocket 更新「处理中」到「成功/失败」。 |

### 1.4 校验与反馈

- 上传前：文件类型、大小（若后端有限制，前端可先校验并提示）。
- 上传中：禁用再次提交、显示进度或 loading。
- 上传成功：message 成功提示，列表自动刷新。
- 上传失败：message 展示服务端返回的错误信息（如格式不支持、超时）。
- 删除失败：message 展示错误信息。

### 1.5 空状态

- 列表无数据时：表格区域显示空状态插画/文案，如「暂无文档，点击上方上传添加」。

## 2. 问答页

### 2.1 目标

用户输入问题，系统基于知识库检索并生成答案，展示答案与引用来源（文档名、chunk 等）。

### 2.2 页面结构

- **顶部/标题**：如「知识库问答」。
- **主内容区**：
  - **输入区**：单行或多行输入框（Input.TextArea），占位符如「请输入您的问题」；右侧或下方「提交」按钮。
  - **回答区**：在提交问题后展示。
    - 上方：答案正文（可富文本或纯文本，按接口返回的 answer 渲染）。
    - 下方：引用区块（如「参考来源」），列表展示每条引用（文档名、可选 chunk 摘要或来源标识）；可折叠/展开。

### 2.3 主要功能细节

| 功能 | 细节 |
|------|------|
| 提问 | 用户输入问题后点击提交；请求问答 API（question）。提交后输入区可禁用或仅允许「清空再问」。 |
| 答案展示 | 请求成功后展示 answer 文本；若接口返回 Markdown，可选用简单 Markdown 渲染。 |
| 引用展示 | 展示 references 列表（文档名、chunk 来源等），与 design 中「引用来源」一致；可每条可点击或仅展示。 |
| 无结果 | 当知识库无相关 chunk 或检索为空时，接口返回友好提示（如「未找到相关内容」）；前端直接展示该提示，不展示答案区块或展示为空状态文案。 |
| 错误 | 网络或服务错误时，message 或页面内提示错误信息，并允许用户重试。 |

### 2.4 校验与反馈

- 提交前：问题为空时禁用提交或点击后 toast 提示「请输入问题」。
- 提交中：按钮 loading、输入区禁用，防止重复提交。
- 提交成功：展示 answer + references；可保留历史问答列表（首版可选，仅展示当前一轮也可）。
- 超时/失败：提示「请求超时」或服务端错误信息，支持再次提交。

### 2.5 空状态与边界

- 首次进入：仅显示输入区，回答区不展示或展示引导文案（如「在上方输入问题开始提问」）。
- 无检索结果：展示接口返回的提示文案（如「知识库暂无相关内容」），不展示答案正文。
- 知识库为空：若全局无文档，可在首问时由接口返回提示，前端统一按「无结果」方式展示。

## 3. 与 tasks 的对应

实现前端时，可与 `tasks.md` 对应如下：

- **文档管理页**：对应 6.1 管理员 REST API（上传、列表、删除）及 2.x 文档处理能力。
- **问答页**：对应 6.2 用户 REST API（question → answer + references）及 4.x 检索匹配、5.x 答案生成能力。
- 组件与交互优先使用 Ant Design（Table、Upload、Input、Button、Message、Modal 等），样式与主题可与 design 保持一致。

## 4. 后续改进

| 功能点 | 说明 | 前端体现 | 建议阶段 |
|--------|------|----------|----------|
| **流式输出（打字机效果）** | 问答接口改为 SSE 或 chunked，前端用 `EventSource` / `fetch` 读流，逐字或逐段追加到 DOM，带光标或动画。 | 流式 I/O、状态与 UI 同步、避免长等待白屏。 | 首版后可做，design 已预留。 |
| **分阶段 loading** | 若后端支持或流式时先返回「检索中」再「生成中」，前端分步提示（如 Steps 或文案切换），减少用户焦虑。 | 对 RAG 流程（检索 → 生成）的理解、状态机。 | 首版或后续。 |
| **多轮对话** | 在上一轮答案下方允许「追问」，前端维护对话列表或传 `conversation_id`，展示多轮问答。 | 状态管理、列表渲染、与后端 session 约定。 | 后续迭代。 |
