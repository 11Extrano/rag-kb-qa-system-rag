## 2. 文档处理

- [ ] 2.1 实现文档上传接口：接收文件或 URL，持久化原始文档并记录 doc_id 与元数据
- [ ] 2.2 实现文本清洗：从文档提取文本，规范化空白与编码，产出清洗后全文
- [ ] 2.3 实现文本拆分：按可配置的 chunk 大小与重叠产出文本片段，并持久化 chunk_id、doc_id、text
- [ ] 2.4 串联流水线：上传完成后自动触发清洗与拆分，并产出可供向量存储消费的 chunk 列表

## 3. 向量存储

- [ ] 3.1 实现 EmbeddingProvider 抽象及至少一种后端（本地或 OpenAI-compatible API）
- [ ] 3.2 接入向量库：实现写入向量与 chunk_id/元数据，支持按 collection/doc 区分
- [ ] 3.3 实现“文本片段 → 向量生成完成”流程：消费文档处理产出的 chunk，调用嵌入并写入向量库
- [ ] 3.4 实现按 query 向量 top-k 检索接口，返回 chunk_id 与相似度分数

## 4. 检索匹配

- [ ] 4.1 实现检索请求接收：用户问答接口入参校验（必填 question），记录检索请求
- [ ] 4.2 将用户 query 转为 embedding，调用向量存储 top-k 检索，取回 chunk_id 与分数
- [ ] 4.3 根据 chunk_id 取回原文与来源元数据，组装为“相关文档匹配完成”的 payload
- [ ] 4.4 无 chunk 时返回空列表并可选提示“知识库为空”

## 5. 答案生成
