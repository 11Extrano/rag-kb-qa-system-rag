## Why

需要基于企业自有文档构建可检索的知识库，并支持用户通过自然语言提问获取基于文档内容的准确答案，避免纯大模型幻觉并保证答案可追溯。当前缺少统一的 RAG 流水线（文档接入 → 向量化 → 检索 → 答案生成），本次变更从零搭建完整系统。

## What Changes

- **文档处理流水线**：支持系统管理员上传文档，自动完成文本清洗、拆分，产出可索引的文本片段（Chunk）。
- **向量存储**：对文本片段做向量化并持久化，为检索提供向量与元数据。
- **检索匹配**：接收用户检索请求，基于向量相似度匹配相关文档/片段，并返回相似度分数。
- **答案生成**：基于检索到的片段与用户问题，生成答案并返回给用户（可含引用来源）。
- **角色与接口**：区分系统管理员（上传、管理）与用户（发起检索、获取答案）。

## Capabilities

### New Capabilities

- `document-processing`: 文档上传、清洗、拆分，产出文本片段；系统管理员触发，系统执行流水线。
- `vector-store`: 文本片段向量化与持久化，支持按向量检索与元数据管理。
- `retrieval-match`: 接收用户检索请求，执行向量检索与相似度匹配，返回相关片段及分数。
- `answer-generation`: 基于检索结果与用户问题生成答案，支持引用来源，面向用户返回。

### Modified Capabilities

- （无：本项目为从零搭建，无既有能力变更。）

## Impact

- **新增**：文档处理服务/模块、向量存储（含嵌入模型与向量库）、检索服务、答案生成服务（含与 LLM 的集成）。
- **依赖**：嵌入模型（或 API）、向量数据库、可选 LLM 服务（答案生成）。
- **接口**：管理员侧文档上传与管理 API；用户侧检索/问答 API 或界面。
