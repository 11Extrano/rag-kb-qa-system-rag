## ADDED Requirements

### Requirement: 系统为文本片段生成向量

系统必须使用配置的嵌入模型或 API 为每个文本片段（来自文档处理）生成向量嵌入，并记录或发出「向量生成完成」。

#### Scenario: 为新片段生成向量

- **WHEN** 文档处理上下文产出了新的文本片段
- **THEN** 系统必须为每个片段计算嵌入，并标记「向量生成完成」

#### Scenario: 嵌入服务失败

- **WHEN** 嵌入服务（本地或 API）对某片段失败或超时
- **THEN** 系统必须记录失败，且不得为该片段持久化无效向量；重试或告警可配置

---

### Requirement: 系统持久化向量与元数据

系统必须存储生成的向量，并将其与片段标识及必要元数据（如 doc_id、chunk_id）关联，以便检索能返回匹配片段。系统必须记录或发出「向量存储完成」。

#### Scenario: 为片段存储向量

- **WHEN** 已为一个或多个片段生成嵌入
- **THEN** 系统必须将向量持久化到向量库，标记「向量存储完成」，并支持后续按向量检索

#### Scenario: 同一片段的幂等写入

- **WHEN** 系统为已存在的片段（相同 chunk_id）存储向量
- **THEN** 系统必须覆盖或 upsert，保证每个片段至多对应一个向量（同一文档/版本下无重复 chunk_id）

---

### Requirement: 系统支持按查询向量检索

系统必须接受查询向量，并返回按相似度（如余弦或内积）排序的已存向量（或片段标识）列表，结果数量有上限（如 top-k）。

#### Scenario: 按查询向量 top-k 检索

- **WHEN** 提供查询向量并指定 top-k
- **THEN** 系统必须从向量库返回最相似的 k 个片段标识（及可选分数）
