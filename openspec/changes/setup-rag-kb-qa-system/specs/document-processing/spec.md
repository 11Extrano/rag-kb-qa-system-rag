## ADDED Requirements

### Requirement: 系统接受管理员上传文档

系统必须接受由系统管理员发起的文档上传，必须持久化原始文档，并发出或记录「文档上传完成」事件。

#### Scenario: 管理员上传单个文档

- **WHEN** 系统管理员通过指定上传接口提交文档（如文件或 URL）
- **THEN** 系统必须存储该文档，并将该事件视为「文档上传完成」

#### Scenario: 拒绝不支持的文件格式

- **WHEN** 系统管理员提交了不支持的文件类型
- **THEN** 系统必须拒绝上传，并返回明确的不支持格式错误信息

---

### Requirement: 系统对上传文本进行清洗

系统必须对已存储的文档内容执行文本清洗（如规范化空白、去除控制字符、可选编码处理），并记录或发出「文本清洗完成」。

#### Scenario: 上传后执行清洗

- **WHEN** 文档已上传并存储
- **THEN** 系统必须对其内容执行文本清洗，并标记「文本清洗完成」

#### Scenario: 清洗后文本为空或无效

- **WHEN** 清洗后提取的文本为空或无效
- **THEN** 系统必须记录失败，且不得对该文档继续执行拆分（或必须向管理员返回错误）

---

### Requirement: 系统将清洗后文本拆分为片段

系统必须按可配置规则（如大小、重叠）将清洗后的文本拆分为文本片段（chunk），并供向量存储使用。系统必须记录或发出「文本拆分完成」。

#### Scenario: 由清洗后文本产出片段

- **WHEN** 某文档的文本清洗已完成
- **THEN** 系统必须将文本拆分为片段，标记「文本拆分完成」，并向下游提供片段（含 chunk id、来源 doc id 等）

#### Scenario: 片段大小与重叠可配置

- **WHEN** 配置中指定了片段大小与重叠（如最大 token 数或字符数）
- **THEN** 系统必须按该参数产出片段
