## ADDED Requirements

### Requirement: 系统接受用户的检索请求

系统必须接受检索请求（如用户问题或查询字符串），并记录或视为「检索请求接收」。请求必须可归属到用户角色。

#### Scenario: 用户以问题发起检索请求

- **WHEN** 用户通过指定接口（如 API 或 UI）提交问题或查询
- **THEN** 系统必须将其作为检索请求接受，并记录「检索请求接收」

#### Scenario: 查询为空或缺失

- **WHEN** 检索请求的查询为空或缺失
- **THEN** 系统必须拒绝该请求并返回校验错误

---

### Requirement: 系统按向量相似度匹配相关文档

系统必须将用户查询转为嵌入（使用与向量库相同或兼容的嵌入模型），对已存向量执行相似度检索，并返回匹配片段及相似度分数。系统必须记录或发出「相关文档匹配完成」。

#### Scenario: 匹配返回带分数的 top 片段

- **WHEN** 已收到有效检索请求且向量库中已索引片段
- **THEN** 系统必须计算查询嵌入、执行相似度检索，返回相关片段（或片段 id）列表及相似度分数，并标记「相关文档匹配完成」

#### Scenario: 向量库中无片段

- **WHEN** 向量库中没有任何片段（如尚未索引任何文档）
- **THEN** 系统必须返回空匹配列表（并可返回知识库为空的提示）
