## Purpose

根据检索到的片段与用户问题，通过 LLM 生成答案并返回给用户；仅依据参考内容回答、支持引用来源，无匹配时不做编造。

## Requirements

### Requirement: 系统根据检索片段与用户问题生成答案

系统必须通过统一抽象（如 LLMProvider）调用配置的 LLM，根据用户问题与「匹配到的内容」列表（含原文、score、文档名等）生成自然语言答案，记录或发出「答案生成完成」，并将答案返回给用户。传给 LLM 的 prompt 须包含：System 约束「仅根据【参考内容】回答，若无法得到答案则说明不编造」；User 为【参考内容】（匹配内容格式化并标注 doc/chunk 来源）+ 用户问题。（MUST）

#### Scenario: 返回带内容的答案

- **WHEN** 用户已提交问题且检索上下文返回了至少一个匹配片段
- **THEN** 系统必须将参考内容与用户问题按上述格式组装为 prompt，调用 LLMProvider 生成答案并返回给用户，并标记「答案生成完成」

#### Scenario: 无匹配片段时的应答

- **WHEN** 检索上下文返回空片段列表
- **THEN** 系统仍须响应用户（如提示未找到相关内容或知识库为空），且不得调用 LLM 在无依据情况下编造内容

---

### Requirement: 系统支持引用来源

系统必须在响应中返回对来源片段的引用（如文档名、chunk_id 或摘要），以便用户知晓答案依据了知识库的哪些部分；引用来自「匹配到的内容」中的 doc/chunk 信息。（MUST）

#### Scenario: 答案包含引用

- **WHEN** 系统返回基于一个或多个检索片段生成的答案
- **THEN** 响应必须包含引用列表（如文档名、chunk 来源等），供用户核实来源；引用与参考内容中的 doc/chunk 标注一致

#### Scenario: 引用在响应格式中可选

- **WHEN** 客户端请求带引用的答案
- **THEN** 系统必须在响应中包含引用列表；若客户端不要求引用，系统可省略或简化

---

### Requirement: 系统可支持流式生成并返回答案

在保留一次性返回答案能力的前提下，系统必须支持「流式生成并返回答案」的可选路径：通过 LlmProvider 的流式调用（如 `stream: true`）逐段获取 LLM 输出，并将答案以流式形式写出；引用在流结束后随元数据或单独事件返回。（MUST）

#### Scenario: 流式生成有匹配时的答案

- **WHEN** 调用方请求流式生成且检索返回至少一个匹配片段
- **THEN** 系统必须按与一次性生成相同的 prompt 约束（仅根据参考内容回答）调用 LLM 流式接口，逐段产出并写出答案文本，最后产出或附带引用列表

#### Scenario: 流式生成无匹配时的提示

- **WHEN** 调用方请求流式生成且检索返回空片段列表
- **THEN** 系统必须输出友好提示（不调用 LLM），并结束流，不编造内容

#### Scenario: 一次性生成行为不变

- **WHEN** 调用方使用现有一次性生成接口
- **THEN** 系统行为与引入流式前一致：返回完整 answer 与 citations，不改变现有 API 契约
